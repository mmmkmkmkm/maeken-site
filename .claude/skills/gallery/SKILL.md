# Gallery: GALLERYへの投稿追加

maeken-siteは静的サイト(GitHub Pages)なので、サイト上に投稿フォームは無い。かわりに前田さんが
Claudeに文章や写真を渡し、Claudeがコードに書き足す、という運用にしている。このスキルはその
「備忘録を投稿に変換する」作業をやりやすくするためのもの。

## いつ使うか

- 前田さんが近況の写真やコメントを送ってきた(明確に「ギャラリーに」と言っていなくても、
  「今日はこんな一日だった」「これ録音風景」のような軽い共有はギャラリー向きの内容であることが多い)
- 「ギャラリーに追加して」「アルバムに載せたい」「サイトに投稿して」と言われたとき
- 判断に迷ったら、追加していいか一言確認すればよい(勝手にpushさえしなければ取り返しはつく)

## 全体の流れ

1. **受け取る**: 前田さんから文章と/または写真をもらう
2. **文章を整える** (必要なら): サイトの他のコピー(About・Contactなど)は短く軽いトーンなので、
   長文であれば1〜3文程度に整理する。本人の言葉のニュアンスは変えず、削るだけに留める
3. **写真があればドット絵化する** (下記参照)
4. `src/data/gallery.ts` の `galleryPosts` 配列の**先頭**に新しい投稿を追加する
   (配列の並び順がそのまま表示順なので、新しい投稿ほど上に来るよう先頭に足す)
5. `npm run build` でエラーが出ないことを確認する
6. **commit/pushは必ず前田さんに確認してから。** 「とりあえず貯めておく」と言われたらローカルの
   変更のまま止め、明示的に「デプロイして」と言われたときだけ commit + push する

## データ形式 (`src/data/gallery.ts`)

```ts
export type GalleryPost = {
  id: string;            // kebab-case、他と被らなければ何でもよい (例: "post-4", "studio-0710")
  type: "photo" | "text";
  date: string;           // "2026.07" のような表記(月まででOK、既存投稿と揃える)
  image?: string;         // type: "photo" のときだけ。相対パス "./assets/gallery/xxx.png"
  caption: string;
};
```

写真投稿は `image` を必須で持たせる。文章のみの投稿は `image` を省略する。

## 写真のドット絵化

送られてきた写真をそのまま置くと、サイトの他の画像(ドット絵ポートレート・コーギースプライトなど)
と質感が揃わない。`scripts/pixelate.py` で軽く加工してから使う:

```bash
python3 .claude/skills/gallery/scripts/pixelate.py <入力画像> public/assets/gallery/<わかりやすい名前>.png
```

- デフォルトでも十分ドット絵っぽくなるが、細部を残したい場合(顔がはっきり写っている等)は
  `--pixels 140` くらいまで上げると顔が潰れすぎない。逆にラフでpop寄りにしたいなら
  `--pixels 80 --colors 24` くらいまで下げる
- `public/assets/gallery/` ディレクトリが無ければ作ってよい
- 出力後は一度 `Read` ツールで画像を見て、潰れすぎ/薄すぎがないか確認してから使う。
  一発で決まらなければ pixels/colors を振って何度か試してよい

写真が既にドット絵っぽい見た目(スクショやAI生成のドット絵)であれば、無理に加工しなくてもよい。
判断に迷ったら両方作って前田さんに見せる。

## 実装例

新しい投稿を追加する編集イメージ(配列の先頭に足す):

```ts
export const galleryPosts: GalleryPost[] = [
  {
    id: "studio-0715",
    type: "photo",
    date: "2026.07",
    image: "./assets/gallery/studio-0715.png",
    caption: "深夜のミックス作業。耳が限界を迎える前に今日は終わり。",
  },
  // ...既存の投稿がこの下に続く
];
```

## 完了後の報告

追加が終わったら、前田さんに次を伝える:
- 何を追加したか(投稿の内容を簡潔に)
- ビルドが通ったこと
- pushはしていないこと(明示的な指示があるまで)、必要なら「デプロイしていい?」と聞く
